# MULTISHOP – Microservices-Based E-Commerce Platform  
## [EN]

MultiShop is a microservices-based e-commerce platform designed to model real-world, enterprise-level e-commerce scenarios.  
The project is developed with a strong focus on scalability, modularity, and modern distributed system principles.

This project is currently **in the early stages of development** and is actively evolving as new services, integrations, and architectural improvements are added.

---

## Project Overview

MultiShop is built from scratch under a single solution and designed as a fully distributed system where each core functionality is handled by an independent microservice.

The platform aims to support different user roles such as authenticated users, guest users, and administrators, each with distinct authorization flows and responsibilities.

---

## Architecture

The project follows **Microservices Architecture** combined with **Onion Architecture** principles at the service level.

Key architectural components include:

- Independent microservices for each domain responsibility
- API Gateway pattern for centralized routing
- Token-based authentication and authorization
- Service-to-service communication patterns
- Scalable and containerized infrastructure

This architecture allows the system to grow incrementally while maintaining clear boundaries between services.

---

## Architectural Patterns & Approaches

The following patterns and approaches are actively used across the project:

- Onion Architecture
- CQRS Design Pattern
- Mediator Pattern
- Repository Pattern
- API Gateway Pattern
- DTO-based data transfer
- Authentication & Authorization strategies

These approaches are applied to address real-world scalability and maintainability challenges.

# MULTISHOP – Mikroservis Tabanlı E-Ticaret Platformu  
## [TR]

MultiShop, gerçek dünya senaryolarını hedefleyen, kurumsal ölçekli bir e-ticaret altyapısını modellemek amacıyla geliştirilen mikroservis tabanlı bir platformdur.  
Proje; ölçeklenebilirlik, modülerlik ve modern dağıtık sistem prensipleri esas alınarak tasarlanmıştır.

Bu proje şu anda **geliştirmenin erken aşamasındadır** ve yeni servisler, entegrasyonlar ve mimari iyileştirmelerle aktif olarak ilerlemektedir.

---

## Proje Genel Bakış

MultiShop, sıfırdan tek bir solution altında inşa edilen ve her temel işlevin bağımsız bir mikroservis tarafından yönetildiği dağıtık bir sistemdir.

Platform; giriş yapan kullanıcılar, misafir kullanıcılar ve yöneticiler için ayrı yetkilendirme senaryolarına sahip olacak şekilde kurgulanmıştır.

---

## Mimari Yapı

Proje genelinde **Mikroservis Mimarisi**, servis seviyesinde ise **Onion Architecture** yaklaşımı benimsenmiştir.

Öne çıkan mimari bileşenler:

- Her domain için bağımsız mikroservisler
- Merkezi yönlendirme için API Gateway yapısı
- Token tabanlı kimlik doğrulama ve yetkilendirme
- Servisler arası iletişim senaryoları
- Konteyner tabanlı ve ölçeklenebilir altyapı

Bu yapı, sistemin adım adım büyümesine olanak tanırken mimari sınırların korunmasını sağlar.
